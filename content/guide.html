Quick start — 4 steps
1
Install Docker Desktop
Required prerequisite. Download from docker.com/products/docker-desktop
Allocate at least 16GB RAM + 4 CPUs in Docker settings.
2
Run the AppImage / DMG
Double-click to launch. No installation needed on Linux (AppImage is portable).
macOS: drag to Applications folder.
3
Complete the setup wizard
Enter mode, LLM keys, cloud/HPC config, data directory. Takes ~2 minutes.
Use this guide for step-by-step help with each section.
4
Click "Boot System" on the Launch page
Docker pulls all ~15 service images from ghcr.io automatically.
First boot: 3–5 min. Subsequent boots: ~30 seconds.
✓ No command line needed  ·  ✓ All images pulled automatically  ·  ✓ Works offline after first run

HPC Configuration
Connect to Slurm · PBS · LSF clusters via SSH + TES protocol
Step 1 — Set up SSH key authentication
OmniBioAI connects to your HPC via SSH. You need passwordless key-based authentication.
# 1. Generate SSH key (skip if you already have one)
ssh-keygen -t ed25519 -C "omnibioai-hpc"

# 2. Copy public key to the HPC cluster
ssh-copy-id username@hpc.university.edu

# 3. Test the connection
ssh username@hpc.university.edu "sinfo"
# Should print partition info without password prompt
Step 2 — Enter HPC connection details in the wizard

Running Your First Workflow
Submit a bioinformatics job in 4 steps
Example — RNA-Seq analysis
1
Open Workflows at :5178
Click the Workflows tile in the Platform Services section of the Workbench.
Browse 18 pre-built bundles: RNA-Seq, WGS, scRNA-seq, ATAC-seq, Metagenomics...
2
Select workflow + click "Register Bundle"
Find RNA-Seq (Nextflow) → click Register Bundle.
Choose engine: WDL, Nextflow, Snakemake or CWL based on your preference.
3
Submit job via TES at :5177
Open TES → select your registered workflow → fill input parameters (FASTQ paths, reference genome).
Choose execution backend: Local Slurm · AWS Batch · Azure Batch · GCP
4
Monitor progress + download results
Track live logs and status in the TES job monitor.
Results written to your configured Work Directory.
✓ Results go to your Work Directory configured in Settings — accessible from both local and HPC shared filesystem.
Frequently Asked Questions
Common issues and solutions
Setup & Installation
?
Docker says "cannot connect to daemon"
Make sure Docker Desktop is running (check the whale icon in your taskbar/menubar). On Linux: sudo systemctl start docker
?
First boot is stuck / taking too long
Docker is pulling ~15 images from ghcr.io. On a slow connection this can take 10–20 min. Check the log panel for progress. Make sure you have a stable internet connection.
?
Port already in use error
Another service is using a required port. Common conflict: system MySQL on 3306 or Redis on 6379.
sudo systemctl stop mysql redis-server then retry.
LLM & AI
?
Dev Hub shows "Backend unreachable"
Make sure the stack is running (Boot System first). Then verify nomic-embed-text model is pulled in Ollama: ollama pull nomic-embed-text
?
Claude API key not working
Check the key starts with sk-ant- and has API access enabled in your Anthropic console. Ensure billing is set up at console.anthropic.com.
HPC & Cloud
?
SSH connection to HPC fails
Test manually: ssh username@hpc.university.edu
Make sure your public key is in ~/.ssh/authorized_keys on the HPC.
?
AWS jobs failing with permission error
Ensure your IAM user has AWSBatchFullAccess and AmazonS3FullAccess policies attached. Check the job queue and compute environment are active in the AWS Console.
